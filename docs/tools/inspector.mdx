[MCP检查器（Inspector）](https://github.com/modelcontextprotocol/inspector)是一款专为测试和调试MCP服务器设计的交互式开发工具。本文将详细介绍检查器的特性和功能，作为[调试指南](/docs/tools/debugging)的补充内容。

## 快速入门

### 安装与基本使用

MCP检查器无需安装，可通过`npx`命令直接运行：

```bash
npx @modelcontextprotocol/inspector <命令>
```

```bash
npx @modelcontextprotocol/inspector <命令> <参数1> <参数2>
```

> 注意：请将尖括号内的内容替换为实际值

#### 检查NPM或PyPi包中的服务器

使用[NPM](https://npmjs.com)或[PyPi](https://pypi.com)包的常见启动方法：

<Tabs>

  <Tab title="NPM包">
  ```bash
  npx -y @modelcontextprotocol/inspector npx <包名> <参数>
  # 示例
  npx -y @modelcontextprotocol/inspector npx server-postgres postgres://127.0.0.1/testdb
  ```
  </Tab>

  <Tab title="PyPi包">
  ```bash
  npx @modelcontextprotocol/inspector uvx <包名> <参数>
  # 示例
  npx @modelcontextprotocol/inspector uvx mcp-server-git --repository ~/code/mcp/servers.git
  ```
  </Tab>
</Tabs>

#### 检查本地开发的服务器

对于本地开发或从代码库下载的服务器，常用的检查方法如下：

<Tabs>
  <Tab title="TypeScript">
  ```bash
  npx @modelcontextprotocol/inspector node 服务器路径/index.js 参数...
  ```
  </Tab>
  <Tab title="Python">
  ```bash
  npx @modelcontextprotocol/inspector \
    uv \
    --directory 服务器路径 \
    run \
    包名 \
    参数...
  ```
  </Tab>
</Tabs>

请务必阅读相关README文件，获取最准确的使用说明。

## 功能概览

<Frame caption="MCP检查器界面">
  <img src="/images/mcp-inspector.png" />
</Frame>

MCP检查器提供了多种与MCP服务器交互的功能模块：

### 服务器连接面板
- 提供选择[传输方式](/docs/concepts/transports)连接服务器的功能
- 支持为本地服务器自定义命令行参数和环境变量

### 资源（Resources）标签页
- 显示所有可用资源列表
- 提供资源元数据查看（MIME类型、描述等）
- 支持资源内容的详细检查
- 内置订阅测试功能

### 提示（Prompts）标签页
- 展示所有可用的提示模板
- 显示提示参数及详细说明
- 允许使用自定义参数测试提示功能
- 可预览生成的消息内容

### 工具（Tools）标签页
- 列出所有可用工具
- 展示工具结构定义和说明
- 支持使用自定义输入进行工具测试
- 直观显示工具执行结果

### 通知面板
- 实时展示服务器记录的所有日志
- 清晰显示从服务器接收的各类通知

## 最佳实践

### 开发工作流程

1. 启动开发
   - 启动检查器并连接到您的服务器
   - 确认基本连接功能正常
   - 检查功能兼容性是否匹配

2. 迭代测试流程
   - 修改服务器代码
   - 重新构建服务器
   - 重新连接检查器
   - 测试修改影响的功能
   - 监控服务器与检查器间的消息交互

3. 异常情况测试
   - 测试无效输入的处理机制
   - 验证缺少必要参数时的反应
   - 进行并发操作测试
   - 检验错误处理和错误响应机制
